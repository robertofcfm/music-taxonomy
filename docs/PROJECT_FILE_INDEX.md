# PROJECT FILE INDEX
Music Genre Taxonomy System

This document describes every file used in the project
and its purpose within the system architecture.

--------------------------------------------------
PROJECT STRUCTURE
--------------------------------------------------

music-taxonomy/

data/
taxonomy/
catalog/
scripts/
reports/
docs/

--------------------------------------------------
DATA DIRECTORY
--------------------------------------------------

data/genre_alias.csv

Purpose:

Stores alias mappings for genre normalization.

Example:

alt rock → Alternative Rock  
synthpop → Synth Pop  

Used during classification to normalize genre names.

--------------------------------------------------
TAXONOMY DIRECTORY
--------------------------------------------------

taxonomy/genre_tree_master.md

Editable taxonomy template.

Characteristics:

• hierarchy defined by indentation  
• contains the official genre structure  
• manually maintained  

This file represents the authoritative taxonomy.

---

taxonomy/genre_tree_operational.csv

Operational representation of the taxonomy.

Contains:

• genre codes  
• parent relationships  
• node types  
• canonical references  

This file is generated from the master taxonomy and
is used by scripts and classification tools.

---

taxonomy/taxonomy_version.md

Stores the version of the taxonomy.

Used to ensure compatibility between:

• taxonomy structure  
• classification results  
• generated genre trees  

--------------------------------------------------
CATALOG DIRECTORY
--------------------------------------------------

catalog/songs_raw.csv

Input file containing the list of songs to classify.

Structure:

title,artist

Example:

Bohemian Rhapsody,Queen

---

catalog/songs_with_genres.csv

Output file generated by the classifier.

Structure:

title,artist,genres,taxonomy_version

Only successfully classified songs are written.

--------------------------------------------------
SCRIPTS DIRECTORY
--------------------------------------------------

scripts/build_tree.py

Script responsible for generating the dynamic
genre tree based on classified songs.

Used in Phase 3.

---

scripts/classify_songs.py

Main classification script.

Responsibilities:

• read songs_raw.csv  
• classify songs into genres  
• validate taxonomy rules  
• output songs_with_genres.csv  

---

scripts/validate_tree.py

Validates taxonomy integrity.

Checks:

• parent-child relationships  
• node type validity  
• taxonomy structure consistency  

--------------------------------------------------
REPORTS DIRECTORY
--------------------------------------------------

reports/taxonomy_issues.csv

Stores issues detected during taxonomy validation.

Examples:

• missing genres  
• structural inconsistencies  
• ambiguous genre relationships  

---

reports/taxonomy_improvement_report.csv

Stores recommendations for improving the taxonomy.

Examples:

• possible merges  
• possible expansions  
• structural improvements  

--------------------------------------------------
DOCUMENTATION DIRECTORY
--------------------------------------------------

High level explanation of the system architecture,
philosophy and components.

---

a new working session.

---

---

---

be respected by the classifier and taxonomy tools.

---

docs/TAXONOMY_RULES.md

Core rules governing taxonomy structure.

---

docs/TAXONOMY_CHANGE_POLICY.md

Defines how the taxonomy may evolve.

---

docs/TAXONOMY_DEPTH_POLICY.md

Defines the minimum depth requirements for the

---

docs/TAXONOMY_NAMING_CONVENTION.md

Defines naming rules for genres.

---

docs/TAXONOMY_QUALITY_CHECKLIST.md

Checklist used to validate taxonomy quality.

--------------------------------------------------
PROJECT PRINCIPLE
--------------------------------------------------

The taxonomy template is the most important file
in the system.

It must never be modified automatically.

All changes must be performed manually by the
project owner.
