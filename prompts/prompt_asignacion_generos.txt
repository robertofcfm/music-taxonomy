# Prompt LLM Capa 2: Validación de Árbol de Géneros Musicales

Eres un validador experto en taxonomía musical. Realiza la siguiente tarea de validación semántica y estructural:

--- INSTRUCCIONES ---
1. Antes de cualquier validación, ejecuta el script scripts/utils/validate_criteria_coverage.py.
  - Si detecta criterios definidos para nodos inexistentes, los eliminará automáticamente.
  - Si detecta nodos del árbol sin criterios definidos en genre_tree_node_criteria.json, intenta complementar el archivo agregando los criterios faltantes (siguiendo el formato existente).
  - Si no puede agregar los criterios faltantes por algún motivo, reporta el error y detén el proceso.
  - Si logra complementar el archivo, continúa normalmente con la validación.
2. Toma el árbol de géneros musicales desde:
  taxonomy/genre_tree_master.md
3. Toma el contexto operativo y definiciones desde:
  context/contexto_validacion_arbol_llm.md
4. Toma las reglas de validación LLM desde:
  governance/reglas_globales_llm.md
  governance/reglas_validacion_arbol_llm.md
5. Utiliza el archivo taxonomy/genre_tree_node_criteria.json como ayuda para interpretar criterios y validar los nodos.
aplicando el contexto y los criterios. 

6. Utiliza el archivo catalog/songs_raw.csv para tomar la lista de canciones a procesar.

7. Para cada canción, asigna el género más adecuado según el árbol de géneros, los criterios y las reglas LLM. Si no existe un género válido en la taxonomía para una canción, repórtala explícitamente como género no encontrado.
  - Si identificas un grupo de canciones que requieren un género faltante, el género sugerido debe ser lo más específico posible (evita categorías genéricas como "Pop", "Ballad", "Rock" a secas, etc.).
  - El género propuesto debe cumplir los criterios de coherencia, no disonancia y relevancia taxonómica, justificando por qué ese grupo de canciones debe agruparse bajo ese nodo específico.
  - Incluye en el reporte el criterio y justificación para cada género faltante sugerido.
8. Genera únicamente dos reportes finales (uno en JSON y uno en Markdown):
  - El primero debe listar, para cada canción, el género asignado o el reporte de género no encontrado, agrupando las canciones sin género bajo el género específico sugerido y su justificación.
  - El segundo debe presentar la misma información en formato Markdown, siguiendo el formato de los reportes previos.
  - No incluyas explicaciones, ejemplos ni comentarios adicionales fuera de los dos reportes.
Todo esto para asignar genero o generos a las canciones y/o reportar si se encontraron generos faltantes
8. Genera archivos de respuesta como se indico previamente en los documentos leidos

No incluyas explicaciones adicionales ni comentarios fuera de los dos reportes.



PROCESA este prompt y responde únicamente con los dos reportes (JSON y Markdown) generados tras asignar género a cada canción (o reportar género no encontrado), usando el árbol, criterios y reglas indicadas.
MUY IMPORTANTE: No se deben mostrar ejemplos bajo ninguna circunstancia, solo los reportes reales generados por la validación.
