# Prompt LLM Capa 2: Validación de Árbol de Géneros Musicales

Eres un validador experto en taxonomía musical. Realiza la siguiente tarea de validación semántica y estructural:

--- INSTRUCCIONES ---
1. Antes de cualquier validación, ejecuta el script scripts/utils/validate_criteria_coverage.py.
  - Si detecta criterios definidos para nodos inexistentes, los eliminará automáticamente.
  - Si detecta nodos del árbol sin criterios definidos en genre_tree_node_criteria.json, intenta complementar el archivo agregando los criterios faltantes (siguiendo el formato existente).
  - Si no puede agregar los criterios faltantes por algún motivo, reporta el error y detén el proceso.
  - Si logra complementar el archivo, continúa normalmente con la validación.
2. Toma el árbol de géneros musicales desde:
  taxonomy/genre_tree_master.md
3. Toma el contexto operativo y definiciones desde:
  context/contexto_validacion_arbol_llm.md
  - contexto_asignacion_generos_llm.md
4. Toma las reglas de validación LLM desde:
  governance/reglas_validacion_arbol_llm.md
5. Utiliza el archivo taxonomy/genre_tree_node_criteria.json como ayuda para interpretar criterios y validar los nodos.
6. Evalúa TU MISMO COMO LLM SIN CREAR ARCHIVOS DETERMINISTICOS cada una de las reglas LLM sobre el árbol, aplicando el contexto y los criterios.
7. Genera dos archivos como respuesta de la evaluacion de TODAS las reglas:
   - Un reporte en JSON con el formato exacto que produce el script de validación determinista (errores, advertencias, ok, detalle, timestamp).
   - Un reporte en Markdown con el mismo formato y secciones que el script (errores, advertencias, reglas validadas, archivos utilizados, fecha).
   Estos se generaran en la carpeta reports con los nombres reporte_validacion_arbol_llm.json y reporte_validacion_arbol_llm.md
   No es necesario mostrarlos en pantalla y los ejemplos son eso ejemplos estos no se incluyen solo lo real

--- FORMATO DE RESPUESTA ---
La respuesta debe contener únicamente:
1. El JSON de reporte de validación.
2. El Markdown de reporte de validación.

No incluyas explicaciones adicionales ni comentarios fuera de los dos reportes.


PROCESA este prompt y responde únicamente con los dos reportes (JSON y Markdown) generados tras evaluar todas las reglas LLM sobre el árbol, usando el contexto y las reglas indicadas.
MUY IMPORTANTE 
no se deben mostrar ejemplos bajo ninguna circunstancia, solo los reportes reales generados por la validación
